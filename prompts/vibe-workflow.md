# Vibecoding 工作流

> 怎么跟 AI 高效协作做游戏。不是教你写代码，是教你怎么让 AI 帮你写。

## 核心原则

1. **截图 > 文字** — 一张参考图顶一千字描述
2. **小步迭代** — 先出最小可玩版本，再逐步加功能
3. **分层拆解** — 把"做一个游戏"拆成 AI 能一次搞定的小任务
4. **不要一口气** — 每次只给 AI 一个明确任务，完成后再给下一个

## 实战流程

### Phase 1: 5 分钟出原型
```
给 AI 的第一个 prompt：

"用 Phaser 3 + TypeScript 做一个最简单的宝可梦风格游戏：
- 一个角色在 800x600 的地图上用方向键移动
- 地图用纯色色块代替（草地=绿色，道路=灰色，水=蓝色）
- 碰到红色区域触发战斗（console.log 就行）
- 不要任何花哨功能，能跑就行
给我一个可以直接打开的 index.html"
```

### Phase 2: 逐个加功能
拿到能跑的原型后，**每次只加一个功能**：
```
// 第 2 个 prompt
"在现有代码上加 NPC：地图上放 3 个 NPC 精灵，按 E 键对话，对话内容用数组存"

// 第 3 个 prompt
"把对话改成打字机效果，底部弹出宝可梦风格对话框"

// 第 4 个 prompt
"加回合制战斗：碰到红色区域进入战斗场景，4 个技能按钮，点击打一下"

// 第 5 个 prompt
"给战斗加 HP 条动画和伤害数字飘字"
```

### Phase 3: 换皮美化
功能都有了，最后统一换美术资源：
```
"我有一套 sprite sheet（附图），帮我替换掉所有色块占位符，
角色用第 1 行做行走动画，NPC 用第 2 行静止帧"
```

## 常见坑和解法

### ❌ "帮我做一个宝可梦游戏"
AI 会给你一个不完整的庞然大物，跑不起来。

### ✅ 拆成 10 个小 prompt
每个 prompt 都能独立验证结果。

### ❌ 用文字描述 UI 布局
"我想要一个左边是头像右边是对话框下面是选项的界面"— AI 猜不准。

### ✅ 截图 + "照这个做"
找一张你想要的 UI 截图，贴给 AI：
"照这张截图的布局实现，用 HTML+CSS"

### ❌ 同时改多个系统
"帮我改战斗系统，顺便把背包也做了，还有地图加个小地图"

### ✅ 一次一件事
改完战斗 → 测试 → 提交 → 再做背包 → 测试 → 提交

## AI 选型

| 任务 | 推荐 AI | 原因 |
|------|---------|------|
| 写游戏逻辑 | Claude / Cursor | 长代码质量最好 |
| UI 布局 | Claude + 截图 | 多模态理解好 |
| 像素美术 | PixelLab | 专门做游戏素材 |
| 立绘生成 | Banana Pro / Midjourney | 高质量角色图 |
| 音效 | Suno / Udio | AI 生成背景音乐 |
| 快速原型 | tldraw Make Real | 画草图直接变代码 |
| 第二意见 | CCG 多 Agent | 三个 AI 互审 |

## 时间分配建议（24 小时比赛）

| 时间 | 任务 |
|------|------|
| 0-1h | 确定创意 + Fork 模板 + 跑通 |
| 1-4h | 核心玩法（移动+战斗+捕捉）|
| 4-6h | 对话系统 + 3 个 NPC |
| 6-8h | UI 美化（RPGUI + 对话框）|
| 8-10h | AI 生成美术资源替换 |
| 10-12h | 音效 + 特效 + 打磨 |
| 12h+ | 测试 + 修 bug + 录 demo 视频 |

## 终极技巧

> **先做 demo 视频再做功能。**
>
> 评委看的是 30 秒 demo，不是你的代码。
> 把最炫的部分先做出来，录 demo，剩下时间再补。
