# Game UI from Reference — AI 游戏界面生成模板

> 给 AI 一张截图，它还你一个能跑的游戏 UI 组件。专治"AI 不理解我要什么"。

## 使用方法

1. 截一张你想要的游戏 UI 参考图（宝可梦对话框、背包界面、战斗HUD、什么都行）
2. 把下面的 prompt 和截图一起发给 Claude / Cursor / GPT-4o
3. 拿到代码，直接用

---

## Prompt 模板（复制即用）

```
你是一个专精游戏 UI 的前端工程师。我会给你一张游戏界面截图作为参考。

## 你的任务
根据截图中的 UI 元素，用 [HTML+CSS+JS / Phaser 3 / React — 选一个] 实现一个功能完整的游戏 UI 组件。

## 要求
1. **像素级还原视觉风格**：圆角、阴影、渐变、字体大小、颜色、间距——全部从截图中提取
2. **响应式**：支持 16:9 和 4:3 两种比例，移动端可点击
3. **动画**：对话框要有打字机效果（逐字显示），菜单要有选中高亮动画
4. **可配置**：文本内容、角色名、头像、选项列表全部通过参数传入，不要硬编码
5. **无外部依赖**：不依赖 UI 框架（除非我指定了），纯 CSS 实现视觉效果
6. **输出结构**：
   - 一个可独立运行的 HTML 文件（或组件文件）
   - 关键样式变量提取为 CSS 变量，方便换肤
   - 简短的使用说明（3 行以内）

## 游戏 UI 类型识别
根据截图自动识别是以下哪种，并应用对应模式：

| 类型 | 关键特征 | 实现要点 |
|------|----------|----------|
| 对话框 | 底部文本框+角色头像+名字 | 打字机效果、点击/空格翻页、支持选项分支 |
| 战斗 HUD | HP条+技能按钮+精灵状态 | 血条动画、技能冷却、状态图标 |
| 背包/菜单 | 网格布局+物品图标+描述 | 拖拽排序、悬停详情、分类标签页 |
| 地图/导航 | 小地图+标记点+方向指示 | 缩放、标记交互、当前位置高亮 |
| 商店 | 物品列表+价格+买卖按钮 | 数量选择、余额检查、交易确认弹窗 |
| 进化/融合 | 前后对比+特效+确认 | 变身动画、属性变化对比、粒子效果 |
```

## 进阶用法

### 批量生成整套 UI
```
我要做一个宝可梦风格的页游，需要以下 UI 组件，每个都给我一个独立文件：

1. 对话框（底部，带角色头像和打字机效果）
2. 战斗界面（回合制，4技能选择）
3. 精灵状态面板（HP/经验值/属性）
4. 背包系统（道具+精灵两个标签页）
5. 精灵捕捉动画（球飞出→摇晃→成功/失败）

参考风格：[贴一张你喜欢的风格截图]
统一配色：[主色 #xxx，强调色 #xxx]
技术栈：纯 HTML+CSS+JS，每个组件可独立引入

所有组件共用一套 CSS 变量体系，确保视觉一致性。
```

### 和 AI 立绘系统联动
```
这个 UI 组件需要支持动态图片加载：
- 精灵头像/立绘通过 URL 传入（来自 Banana Pro API 实时生成）
- 图片加载时显示骨架屏占位
- 支持 webp/png，自动适配尺寸
- 立绘加载失败时显示默认剪影
```

### 对话系统深度定制
```
实现一个完整的游戏对话引擎：

功能：
- 打字机效果（可调速度，点击跳过）
- 支持富文本：颜色标记 {red}危险{/red}、抖动 {shake}地震{/shake}、大字 {big}BOSS{/big}
- NPC 头像 + 表情切换（同一对话中表情可变）
- 选项分支（选完继续不同对话线）
- 自动换行 + 翻页（文本超出框自动分页，显示 ▼ 提示）
- 全局对话历史（按 L 键查看历史记录）

数据格式用 JSON：
{
  "id": "npc_001_greeting",
  "speaker": "老橡树博士",
  "portrait": "oak_smile.png",
  "lines": [
    {"text": "欢迎来到精灵世界！", "emotion": "smile"},
    {"text": "这里危险又{red}刺激{/red}...", "emotion": "serious"},
    {
      "text": "你准备好了吗？",
      "choices": [
        {"text": "准备好了！", "next": "npc_001_ready"},
        {"text": "再等等...", "next": "npc_001_wait"}
      ]
    }
  ]
}
```

## Tips

- **一张好的参考图 > 一千字描述**。别试图用文字解释 UI 长什么样，截图贴上去
- **分步迭代**：先让 AI 出基础版，看到效果再提修改，别一次塞太多需求
- **CSS 变量是换肤神器**：让 AI 把所有颜色/圆角/字号抽成变量，后面改风格改变量就行
- **AI 生图 + AI 写 UI = 全 AI pipeline**：Banana Pro 生精灵立绘，Claude 写 UI 组件，人类只管设计和玩
