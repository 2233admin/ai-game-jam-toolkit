# CCG 共识门控规则 (Consensus Gate)

> 复制这段到你的 AI 工具（Cursor Rules / Claude CLAUDE.md / System Prompt）

## 规则

当你需要做重要的技术决策时，遵循以下共识流程：

### Step 1: 独立求解
将同一问题发给至少 2 个 AI（推荐 3 个）：
- **AI-A** (例如 Claude): 擅长架构、代码质量、长文件编辑
- **AI-B** (例如 GPT/Codex): 擅长快速实现、广泛知识
- **AI-C** (例如 Gemini): 擅长搜索、大上下文、多模态

### Step 2: 共识分类

| 分类 | 条件 | 行动 |
|------|------|------|
| ✅ **AGREE** | 三方方向一致，实现细节兼容 | 直接执行综合方案 |
| ⚠️ **SPLIT** | 方向一致但实现分歧（不同技术栈/架构） | 列出分歧 + 选一个 + 给理由 |
| ❌ **CONFLICT** | 根本方向冲突（一方说做一方说不做） | 停下来，人工决策 |

### Step 3: 输出格式

```markdown
## Consensus: [AGREE|SPLIT|CONFLICT]

### AI-A 观点
[核心建议]

### AI-B 观点
[核心建议]

### AI-C 观点（如果有）
[核心建议]

### 综合结论
[最终方案 + 选择理由]

### 分歧点 (SPLIT/CONFLICT 时)
- [分歧1]: A 说 X，B 说 Y → 选 X 因为 Z

### Action Items
- [ ] ...
```

### Fallback

如果只有一个 AI 返回结果，标注为 `PARTIAL` 而非 AGREE，提醒缺少视角。

## 什么时候该用 CCG？

| 场景 | 用不用 |
|------|--------|
| 架构决策（选 Phaser 还是 Kaplay） | ✅ 用 |
| 复杂 bug 排查 | ✅ 用 |
| 性能优化方案 | ✅ 用 |
| 简单 CSS 调整 | ❌ 没必要 |
| 写一个工具函数 | ❌ 没必要 |
| 重构整个战斗系统 | ✅ 用 |

## Cursor Rules 版本

如果你用 Cursor，把以下内容放到项目根目录的 `.cursorrules`：

```
当我问你技术选型或架构问题时，请：
1. 先给出你的方案
2. 然后假设你是另一个持不同意见的 AI，给出反对意见
3. 最后综合两个视角，给出最终推荐

格式：
## 我的方案: ...
## 反方观点: ...
## 最终推荐: ... (选了哪个，为什么)
```
